# Vivea — Product Principles

## Principle 1: Never block logging
Do not block someone from logging when they 
may be impaired, scared, post-seizure, or 
trying to capture something quickly.

This is the foundational principle that drives 
every product decision in Vivea.

The critical moment is not "user acquisition." 
The critical moment is: "I need to record this 
before I forget."

Users may be dealing with:
- Memory gaps post-seizure
- Confusion or disorientation
- Fatigue during recovery
- Caregiver input on behalf of someone else
- Limited time before details fade

Every friction point — signup walls, required 
fields, complex forms — costs clinical data 
that could improve someone's care.

### What this means in practice
- No signup wall on first launch
- No required fields except event type and time
- Capture now, enrich later
- Account creation comes after value is delivered
- Local storage by default, account optional

### The onboard-first model
Start tracking → understand value → create account

Not: Create account → onboard → start tracking

Account creation is tied to clear value moments:
- After first log: "Save this securely?"
- After 3+ logs: "You've logged 3 events. 
  Create an account so you don't lose your history."
- Before export: "Create an account to generate 
  doctor-ready reports."

### Local-only transparency
Until an account is created, show clearly:
"Your data is saved on this device only."
This avoids a trust issue later and respects 
the user's informed choice.

## Principle 2: Finite problems get deterministic 
## solutions. Infinite problems get AI.

Not every feature is an AI problem.

A finite problem has a known output space — 
chip selectors, conditional forms, settings, 
navigation. Faster, cheaper, more reliable, 
works offline.

An infinite problem has an unbounded output 
space — pattern recognition, natural language, 
narrative synthesis, open-ended questions. 
This is where AI earns its place.

Ask before adding AI to any feature:
*Is the output space finite or infinite?*

## Principle 3: Personalization reduces work
The app should get smarter with every piece 
of information it learns about the user.

Onboarding data immediately reduces friction:
- Known seizure type → pre-selected in log form
- Known medications → shown in today's med schedule
- Known triggers → pre-populated as chips
- Caregiver mode → language shifts throughout

The user should never have to tell Vivea 
something twice.

---

## Principle 4: Clinical credibility without 
## clinical coldness

Vivea handles serious medical data but speaks 
to humans, not patients.

- Warm, calm language throughout
- No medical jargon in the UI
- Educational context for newly diagnosed users
- Never catastrophize — seizure logged, 
  rest now, you're safe

---

## Principle 5: Every log should mean something

Data without synthesis is a diary. 
Vivea turns logs into intelligence.

Every log entry feeds:
- Pattern Agent → insights the user can act on
- Clinical Agent → summaries doctors can use
- Navigation Agent → answers to care questions

The goal is not to collect data. 
The goal is to help people live better.
